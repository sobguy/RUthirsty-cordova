üîç Testing localStorage

View Stored Data

  1. Open browser DevTools (F12)
  2. Go to Application tab (Chrome) or Storage tab (Firefox)
  3. Expand "Local Storage" in left sidebar
  4. Click on "http://localhost:8000"
  5. Find key: waterIntakeRecords
  6. View the JSON array of records

Manual Data Manipulation

In browser console, try these commands:

// View all records
localStorage.getItem('waterIntakeRecords')

// Clear all data
localStorage.removeItem('waterIntakeRecords')
location.reload()

// Add test data
DataManager.addRecord()
UIController.updateDisplay()

üêõ Common Issues & Solutions

Issue Solution
Button doesn't respond Check browser console for JavaScript errors. Ensure all JS files loaded correctly.
Counter doesn't update Verify DataManager and UIController are loaded. Check console for errors.
Data doesn't persist Check if localStorage is enabled in browser. Some private/incognito modes disable it.
Styles not loading Verify style.css path is correct. Check Network tab in DevTools.
cordova.js 404 error Normal in browser mode. Cordova.js only exists in built apps.

üì± Android Testing

Prerequisites

  • Android SDK installed
  • ANDROID_HOME environment variable set
  • Java JDK installed (version 11 or higher)
  • Gradle installed (or use wrapper)

Build APK

cd /workspaces/RUthirsty-cordova/ruthirsty
cordova requirements # Check setup
cordova build android # Build APK

# APK location:
# platforms/android/app/build/outputs/apk/debug/app-debug.apk

Test on Device

# Enable USB debugging on Android device
# Connect device via USB
adb devices # Verify device connected
cordova run android --device

Test on Emulator

# Create emulator in Android Studio or:
avdmanager create avd -n test -k "system-images;android-30;google_apis;x86_64"
emulator -avd test # Start emulator
cordova emulate android
